# Handoff — простым языком (C1 PE дизайн)

**Date:** 2026-07-15  
**Status:** дизайн на компьютере — GO · реактивы / клетки — NO-GO  

Произошло **не редактирование ДНК**, а успешная подготовка дизайна к возможному эксперименту.

---

## По шагам

Сначала выбрали кандидат **C1** и конкретное изменение:

```text
A → G
```

Обычный base editor для этой позиции неудобен: нужная буква плохо попадает в его рабочее окно. Поэтому выбрали **prime editing** — систему, которая может точнее записать нужную замену.

Затем нашли основной дизайн **PD1**:

* он направляет редактор к C1;
* содержит инструкцию, какую букву записать;
* для повышения эффективности подобрали дополнительный `ngRNA`.

После этого последовательности проверили через CRISPOR. Это предварительный тест специфичности: программа ищет другие участки генома, куда направляющая потенциально может ошибочно привязаться.

Результат оказался приемлемым:

* почти идентичных опасных совпадений не найдено;
* общая специфичность хорошая;
* найдено несколько менее похожих потенциальных off-target участков;
* наиболее важный из них — около или внутри **RADIL** (**кандидат на обязательную проверку**, не доказанный off-target);
* для вспомогательного `ngRNA` найден вариант с ещё лучшей специфичностью.

---

## Что это означает

Текущий статус:

```text
дизайн на компьютере — GO
заказ реактивов и работа с клетками — ещё NO-GO
```

То есть конструкция выглядит достаточно хорошей, чтобы продолжать планирование. Но компьютерный прогноз не гарантирует, что:

* редактирование реально произойдёт;
* оно будет достаточно эффективным;
* не появятся побочные изменения;
* RADIL и другие потенциальные участки останутся нетронутыми.

В лаборатории пришлось бы отдельно секвенировать:

```text
C1
RADIL
другие приоритетные off-target loci
```

## Почему появились три ветки

### Ветка A — проверить 3D-архитектуру

Вопрос:

> Меняет ли замена C1 контакт между enhancer и promoter?

Для этого нужно:

```text
отредактировать C1
→ подтвердить правильный edit
→ измерить контакт Capture-C
→ сравнить REF и ALT
```

Это главная ветка для проверки архитектурной гипотезы, но она сложнее и дороже.

### Ветка B — проверить обычную регуляторную активность

Берётся короткий фрагмент около C1, например 301 bp:

```text
REF sequence
против
ALT sequence
```

и сравнивается их активность в reporter assay.

Это быстрее и дешевле, но отвечает на другой вопрос:

> Изменяет ли буква локальную способность участка управлять транскрипцией вне полноценной 3D-структуры?

Интерпретация:

```text
reporter изменился
→ возможен обычный activity-механизм

reporter не изменился
→ activity-effect не обнаружен,
  но 3D-механизм всё ещё возможен
```

Reporter **не** подтверждает и **не** исключает изменение петли сам по себе.

### Ветка C — остановиться

Можно сохранить C1 как вычислительного кандидата и ничего не заказывать, пока не появятся:

* лабораторный партнёр;
* бюджет;
* приоритет относительно других кандидатов;
* окончательный GO-документ.

## В одной схеме

```text
C1 A→G выбран
→ base editing неудобен
→ разработан prime-editing дизайн PD1
→ CRISPOR не нашёл критических близких off-target
→ RADIL отмечен для обязательной проверки
→ дизайн пригоден для планирования
→ биологический эффект ещё не проверен
```

**Главный итог:** найден технически правдоподобный способ создать вариант C1, но пока ничего не доказано о его влиянии на активность, контакт хроматина или экспрессию.
